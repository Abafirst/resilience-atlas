# IATLAS Activity Video Guidelines

## Video Specifications
- **Length**: 2–5 minutes per video
- **Format**: MP4, H.264 encoding
- **Resolution**: 1080p (1920×1080)
- **Aspect ratio**: 16:9
- **Captions**: Required (SRT file submitted alongside video)
- **Audio**: Clear narration, minimal background noise, 48kHz stereo

## Content Structure

Each video follows this structure:

1. **Introduction** (15 sec) — Activity name, dimension, age group
2. **Materials overview** (20 sec) — Show all required materials
3. **Step-by-step demonstration** (2–4 min) — Clear, paced instructions
4. **Age-group variations** (30 sec) — Modifications for other age groups
5. **Facilitation tips** (20 sec) — Practitioner guidance
6. **Closing** (15 sec) — Recap and encouragement

## Accessibility Requirements

- **Closed captions**: All videos require SRT caption files
- **Visual descriptions**: Key on-screen actions narrated aloud
- **Pacing**: Allow sufficient time between steps for practitioners/parents to follow along
- **Font size**: Any on-screen text must be minimum 24pt
- **Contrast**: All text overlays must meet WCAG 2.1 AA contrast requirements

## Representation Guidelines

- Include diverse facilitators (age, gender, ethnicity, ability)
- Show children of diverse backgrounds completing activities
- Use gender-neutral language throughout
- Avoid assumptions about family structure

## Technical Submission Requirements

Each video submission must include:
- MP4 video file (H.264)
- SRT captions file
- Thumbnail image (1280×720 JPG)
- Video metadata form (title, dimension, age groups, materials list)

## Naming Convention

`IATLAS_[dimension-key]_[activity-id]_[age-group]_v[version].mp4`

Example: `IATLAS_somatic-regulative_box-breathing_ages-8-10_v1.mp4`

## Production Workflow

1. Script approval by clinical team
2. Video production
3. Caption file creation
4. Clinical review
5. Accessibility review
6. Final approval
7. Upload to video hosting platform
8. Update `videoUrl` field in activity data

## Hosting

Videos will be hosted on a dedicated video platform. The `videoUrl` field in each activity's data object should be updated with the final URL once production is complete.
